Per Recital (56), AI systems for determining access/admission, assigning persons to programmes, evaluating learning outcomes, assessing education levels, and monitoring prohibited student behaviour during tests are high-risk. However, per Recital (53), an educational AI system would not be high-risk if it: performs only narrow procedural tasks (e.g., classifying incoming student documents), improves previously completed human work (e.g., improving language in already-written feedback), detects deviations from prior patterns (e.g., flagging grading inconsistencies after teacher assessment), or performs preparatory tasks (e.g., indexing student files). Critically, these exceptions are overridden if the system involves profiling under Regulation (EU) 2016/679.
