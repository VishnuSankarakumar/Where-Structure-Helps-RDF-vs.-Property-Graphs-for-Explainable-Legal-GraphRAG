This project covers a comparative study of Neo4j Labelled Property Graphs and RDF triplestores as backends for Retrieval Augmented Generation. A Neo4j graph is built from the EU AI Act using an existing LLM Graph Builder tool, then reverse-engineered into a formal RDF ontology through a custom transformation pipeline involving statistical analysis of the Neo4j schema, a human-in-the-loop mapping workflow covering 3,816 entities and 6,552 relationships, and a Python transformation script with zero data loss. Both systems are evaluated on 50 questions using RAGAS and general qualitative human evaluation. Neo4j outperforms on overall scores and on simple factual questions, where its targeted retrieval and LLM-friendly return format give faithful answers. However, RDF achieves higher context recall on complex questions, where ontology-driven retrieval finds information that Neo4j's vector-based localisation misses.
